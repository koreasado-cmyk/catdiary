# 키코다이어리 개발 기록 🐈

> 고양이 건강 다이어리 + 병원 리포트 웹앱이 어떻게 만들어졌는지 정리한 문서입니다.

---

## 1. 무엇을 만들었나

집냥이·길냥이를 카테고리별로 등록하고, **매일 30초 버튼식 건강 기록 → 캘린더 → 통계 → 병원 제출용 리포트**까지 한곳에서 관리하는 모바일/아이패드/PC 웹앱입니다.

처음엔 단순한 "고양이 다이어리"였지만, 시중 반려동물 건강앱(PetDesk·멍냥보감·집사일기·11pets 등)을 분석해서 **고양이 전용 + 병원 리포트 + 30초 기록**이라는 차별점을 가진 건강 다이어리로 발전시켰습니다.

---

## 2. 파일 구성

| 파일 | 역할 |
|------|------|
| `index.html` | 화면 껍데기(HTML/CSS) + 시드 데이터·사진이 주입된 셸 |
| `app.js` | 앱의 모든 로직 (약 57KB) — 화면 그리기·기록·통계·리포트·동기화 |
| `README.md` | 사용/배포 설명서 |
| `개발기록.md` | 이 문서 |

> ⚠️ **두 파일(`index.html` + `app.js`)을 반드시 함께 올려야** 동작합니다.

### 왜 두 파일로 나눴나?
처음엔 HTML 한 파일에 다 넣었는데, 기능이 늘면서 코드가 너무 커졌습니다.
그래서 **로직을 `app.js`로 분리**하고 `index.html`에서 `<script src="app.js">`로 불러오도록 구조를 바꿨습니다. 화면(HTML)과 로직(JS)이 분리돼 수정·관리가 쉬워졌습니다.

### 빌드 방식 (개발용)
`build_diary_app.py`(상위 폴더)가 시드 고양이 사진 4장(코코·푸링·밀크·레몬)을 720px로 리사이즈해서 base64로 `index.html`에 심고, `app.js`를 배포 폴더(`catdiary/`)로 복사합니다. → Python + Pillow 필요.
(이미 빌드된 결과물이 이 폴더이므로, 일반 사용/수정 시엔 빌드를 다시 돌릴 필요 없습니다.)

## 3. 핵심 기능

- **첫 화면**: 카테고리(집냥이/길냥이) + "📝 오늘 기록" 할일 + "⚠️ 관찰 필요" 알림 + 생일/기념일 D-day
- **고양이 프로필**: 사진·생일·성별·중성화·품종·몸무게·질병/알레르기·다니는 병원·관심관리
- **30초 건강기록**: 컨디션·식사·물·대변·소변·구토·놀이·몸무게·약·사진·메모를 **버튼으로 톡톡** (같은 날은 자동 이어쓰기)
- **캘린더**: 구토(빨강)·묽은변(노랑)·약(파랑)·병원(보라)·사진(초록) 마커
- **통계**: 몸무게 변화 그래프(SVG) + 최근 30일 요약
- **병원 리포트**: 최근 7일/30일 요약 → 🖨 인쇄로 PDF 저장 (헤더·탭·버튼은 인쇄 시 자동 숨김)
- **병원 기록**: 방문일·증상·진단·처방·진료비·다음 방문일
- **기념일** D-day, **언어 전환**(한국어/일본어)
- **저장**: Firebase 클라우드 동기화 또는 이 기기 저장(localStorage) 자동 선택 + 백업 내보내기

### 데이터가 저장되는 방식
- 기본은 **이 기기에만 저장**(localStorage) — 기기마다 따로 보관됩니다.
- 설정에서 **Firebase** 값을 넣으면 폰·아이패드·PC가 **실시간 동기화**됩니다. 사진은 용량 때문에 별도 보관(`rooms/{방코드}/photos`)하고, 본문 데이터만 작게 유지합니다.
- 가족이 같은 **공유 코드(방 코드)** 를 쓰면 데이터를 함께 봅니다.

## 4. GitHub 업로드 & 배포 과정

실제로 이 폴더를 인터넷에 올린 순서입니다.

1. **GitHub CLI 설치**: `winget install GitHub.cli`
2. **로그인**: `gh auth login` — 브라우저에서 일회용 코드 입력 방식으로 본인 계정 인증
3. **Public 저장소 생성 + 업로드**:
   ```
   gh repo create catdiary --public --source . --remote origin --push
   ```
   → 저장소: **https://github.com/koreasado-cmyk/catdiary**
4. **GitHub Pages 켜기** (실제 접속 가능한 앱 링크 생성):
   ```
   gh api -X POST repos/koreasado-cmyk/catdiary/pages -f "source[branch]=main" -f "source[path]=/"
   ```
   → 앱 주소: **https://koreasado-cmyk.github.io/catdiary/**

이후 코드를 고치면 `git commit` → `git push` 하면 사이트가 1~2분 뒤 자동 업데이트됩니다.

## 5. 앱 이름 변경 작업 (냥로그 → 키코다이어리)

첫 화면 상단 제목을 바꾼 과정입니다.

- 제목은 `app.js`의 다국어 사전 `appTitle` 값으로 표시됩니다 (`$('#title').textContent = t('appTitle')`).
- **한국어**: `냥로그 🐈` → `키코다이어리 🐈`
- **일본어**: `ニャンログ 🐈` → `キコダイアリー 🐈`
- JS 로딩 전 잠깐 보이는 정적 제목(`index.html`의 `<div id="title">`)도 함께 변경.
- 소스(상위 폴더 `_app_template.html`/`app.js`)와 배포본(`catdiary/`) **양쪽 모두** 수정 → 프리뷰로 확인 → `git push`.
