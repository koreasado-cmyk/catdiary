# 냥코수첩 · 고양이 건강 다이어리 🐈

카테고리(집냥이/길냥이…)별로 고양이를 등록하고, **매일 30초 건강 기록 + 일상 다이어리 +
병원에 보여줄 수 있는 요약 리포트**를 한곳에서 관리하는 모바일/아이패드/PC 웹앱입니다.
화면은 `index.html`, 로직은 `app.js` 두 파일로 동작합니다. (둘 다 올려야 합니다)

## 기능
- 첫 화면 **카테고리**(집냥이/길냥이) + **오늘 기록** 바로가기 + **관찰 필요** 알림
- 고양이 **프로필**: 사진·생일·성별·중성화·품종·현재 몸무게·질병/알레르기·다니는 병원·관심관리
- **30초 건강기록**: 컨디션·식사·물·대변·소변·구토·놀이·몸무게·약·사진·메모(버튼식, 같은 날은 자동 이어쓰기)
- **캘린더**: 구토(빨강)·묽은변(노랑)·약(파랑)·병원(보라)·사진(초록) 마커로 한눈에
- **통계**: 몸무게 변화 그래프 + 최근 30일 요약(구토·묽은변·식욕저하·물부족·병원·체중변화)
- **병원 제출용 리포트**: 최근 7일/30일 요약 → 🖨 인쇄로 **PDF 저장** 가능
- **병원 기록**: 방문일·증상·진단·처방·진료비·다음 방문일
- **기념일** 등록 → 홈에 D-day로 표시(생일 자동 포함)
- **언어** 한국어/일본어 전환(설정)
- **클라우드 동기화**(Firebase) 또는 **이 기기 저장**(localStorage) 자동 선택 + 데이터 **백업 내보내기**

> 📄 PDF 저장: 리포트 화면 하단 **🖨 인쇄 / PDF 저장** → 인쇄 대화상자에서 "PDF로 저장" 선택
> (헤더·탭·버튼은 인쇄 시 자동으로 숨겨지고 요약만 깔끔하게 출력됩니다)

## 1) GitHub Pages에 올리기 (HTML 링크 만들기)

1. GitHub에서 새 저장소 생성 (예: `catdiary`) — Public 권장
2. 이 폴더(`catdiary`)를 저장소에 올립니다:

   ```bash
   cd catdiary
   git init
   git add index.html app.js README.md
   git commit -m "냥로그 고양이 건강 다이어리"
   git branch -M main
   git remote add origin https://github.com/<내아이디>/catdiary.git
   git push -u origin main
   ```

3. 저장소 → **Settings → Pages** → Source를 **main / (root)** 로 설정 후 저장
4. 1~2분 뒤 발급되는 주소로 접속:
   `https://<내아이디>.github.io/catdiary/`
   → 이 링크를 폰·아이패드 홈 화면에 추가하면 앱처럼 실행됩니다.

> `index.html`과 `app.js`를 **둘 다** 올려야 동작합니다. 기본 초기화 버전이라 샘플 고양이 없이 빈 상태로 시작하며, **집냥이/길냥이** 카테고리만 들어 있습니다. 카테고리를 눌러 우리 아이를 직접 등록하세요 🐾

## 2) 어디서나 DB 연동하기 (Firebase, 무료)

GitHub Pages는 서버가 없는 정적 호스팅이라, 기기 간 동기화는 클라우드 DB가 필요합니다.
**Firebase Firestore**를 쓰면 폰·아이패드·PC에서 같은 기록을 실시간으로 봅니다.

### 설정 순서
1. https://console.firebase.google.com 접속 → **프로젝트 만들기**
2. 왼쪽 **빌드 → Firestore Database → 데이터베이스 만들기** → **테스트 모드**로 시작
3. 프로젝트 개요 옆 **⚙ → 프로젝트 설정 → 내 앱 → 웹(`</>`)** 추가
4. 표시되는 `firebaseConfig`에서 아래 값을 복사:
   `apiKey`, `authDomain`, `projectId`, `appId`
5. 앱을 열고 우측 상단 **⚙ 설정** → Firebase 칸에 붙여넣고 **저장 후 적용**
6. **가족 공유 코드(방 코드)**: 가족 모두 같은 값(예: `noh-family`)을 넣으면 데이터가 공유됩니다.

> 설정 전에는 자동으로 **이 기기에만 저장(localStorage)** 모드로 동작합니다.
> 빈 상태로 시작하므로, 등록한 고양이·기록이 그대로 클라우드에 올라갑니다.

### Firestore 보안 규칙 (간단 버전)
테스트 모드는 약 30일 뒤 만료됩니다. 가족만 쓸 거라면 규칙을 이렇게 둘 수 있어요
(방 코드를 비밀로 유지하세요):

```
rules_version = '2';
service cloud.firestore {
  match /databases/{db}/documents {
    match /rooms/{room}/{document=**} {
      allow read, write: if true;
    }
  }
}
```

> 더 안전하게 하려면 로그인(Auth) 연동이 필요합니다. 원하면 추가 가능.

## 3) 아이패드 / 아이폰 호환성

- ✅ **Safari (iPadOS/iOS)** 정상 동작 확인 포인트:
  - 사진 업로드: `FileReader` + `<canvas>` 리사이즈 → HEIC도 선택 시 자동 변환됨
  - 날짜 입력: `input type=date` 네이티브 달력 사용
  - 실시간 동기화: Firebase v10 웹 SDK(ES module) — Safari 11+/iPadOS 지원
  - 안전영역(노치/홈바) `env(safe-area-inset-*)` 반영, 화면 폭 600px 중앙 정렬
- 📲 **홈 화면에 추가**: Safari 공유 → "홈 화면에 추가" → 전체화면 앱처럼 실행(아이콘 🐈)
- ⚠️ 참고: 클라우드(Firebase)를 켜기 전에는 기기별로 따로 저장됩니다.
